# Draft Rulebooks

Active draft-generation rules live here.

- `PV_NAMING_RULEBOOK.md`: SEO_V3 PV identity naming rulebook (skeleton —
  awaiting redrafting after the 2026-06-02 hard-reset alignment).
- `DATABASE_POOL_INPUT_CONVERSION_RULEBOOK.md`: active agent procedure for
  converting natural-language or semi-structured `inputs/<pool_id>/` source
  material into reviewable SEO_V3 `database_pool/<pool_id>/` rows.

Draft agents use these rules to convert source PV material into normalized
SEO_V3 database-pool outputs. While `PV_NAMING_RULEBOOK.md` is a skeleton,
agents must use the standard source document
(`inputs/4GSR_Beamline_PV_Naming_Standard_v1.0/standard.md`) and the
abbreviation registry (`database_pool/abbreviations/registry.json`) as the
working policy and surface unresolved cases through the exception/proposal
workflow.
